VPC ibm_is_placement_group

Module: ibm_is_placement_group
Category: VPC Infrastructure
Version: 1.0.0

Description

Manage IBM Cloud Ibm Is Placement Group resources. This module supports creating, updating, and deleting resources with full idempotency support.

Requirements

  • Python 3.10+
  • Ansible Core 2.14+
  • IBM Cloud SDK Core 3.20.0+
  • IBM Cloud Python SDK (vpc or platform-services)

Parameters

Parameter Type Required Description
name string Yes Name of the resource
id string No Resource ID (for updates/deletes)
state string No Desired state: present, absent (default: present)
ibmcloud_api_key string Yes IBM Cloud API key
region string No IBM Cloud region (default: us-south)

Examples

Create Resource

- name: Create Ibm Is Placement Group
  ibm_is_placement_group:
    name: my-resource
    state: present
  register: result

Update Resource

- name: Update Ibm Is Placement Group
  ibm_is_placement_group:
    id: "{{ result.resource.id }}"
    name: updated-name
    state: present

Delete Resource

- name: Delete Ibm Is Placement Group
  ibm_is_placement_group:
    id: "{{ result.resource.id }}"
    state: absent

Check Mode (Dry Run)

- name: Test changes without applying
  ibm_is_placement_group:
    name: my-resource
    state: present
  check_mode: yes

Return Values

Key Type Description
resource dict Resource information including ID, name, and all attributes
changed bool Whether the resource was changed
msg string Status message describing the operation result

Notes

  • This module is idempotent - safe to run multiple times
  • Supports check mode for dry-run testing
  • Uses native IBM Cloud Python SDK
  • No Terraform dependencies required
